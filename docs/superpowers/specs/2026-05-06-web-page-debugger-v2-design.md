# web-page-debugger v2 设计文档

**日期：** 2026-05-06
**状态：** 已批准
**目标：** 升级现有 `web-page-debugger` skill，使 AI 能自主验收和调试 web 产品

---

## 背景

现有 `web-page-debugger` v1.0.0 仅支持被动的 debug 排查，且完全依赖 `chrome-devtools-mcp`。
v2 目标：统一验收 + 调试，支持多 MCP 后端，形成完整的"发现问题 → 报告 → 修复 → 重验"闭环。

---

### Tool Adapter

启动时自动检测可用 MCP 工具：
1. 优先使用 `@playwright/mcp`（多浏览器、自动化能力更强）
2. Fallback 到 `chrome-devtools-mcp`
3. 两者都不可用 → 报错，提示用户安装其中一个

## Verify 模式

### 触发

```
/web-page-debugger verify <url> [spec或描述]
```

### 验收标准来源（优先级）

1. **项目 spec 文件**：自动扫描 `docs/`、`*.md`、`PRD*` 等文件
2. **用户口头描述**：直接解析成可执行的验收项
3. **AI 自主推断**：导航页面后，根据 UI 元素推断预期行为

### 执行流程

```
Step 1  打开页面，截图建立 baseline
Step 2  确定验收标准来源，生成验收清单
        → 先展示给用户确认（可增删），再执行
Step 3  逐项执行：点击、填表、导航、断言响应
Step 4  每项记录结果：✓ 通过 / ✗ 失败（附截图 + 失败原因）
Step 5  输出 CI 风格总结报告
Step 6  有失败项 → 报告 + 修复建议，等待用户指令
```

**关键设计：** 生成验收清单后，必须先给用户确认再执行，防止 AI 推断错误浪费时间。

## Debug 模式

### 触发

```
/web-page-debugger debug <url> [症状描述]
```

### 症状驱动的排查优先级

| 症状关键词 | 优先检查 |
|-----------|---------|
| 白屏、blank | JS bundle 加载 → console error |
| 登录、401、403 | Network auth → cookie/token |
| 慢、loading | Performance trace → TTFB/LCP |
| 报错、crash | Console error → stack trace |
| 布局错乱 | Screenshot → computed style |
| 无症状描述 | 默认：console → network → DOM |

### 执行流程

```
Step 1  截图记录现象
Step 2  根据症状选择排查顺序
Step 3  逐层深入（console → network → DOM/JS）
Step 4  定位根因后立即停止，不过度排查
Step 5  输出：根因 + 证据链 + 复现路径 + 修复建议
Step 6  等待用户指令：确认修复 → 进入 Repair Loop
```

---

## Repair Loop

### 触发条件

用户在报告后明确表示修复意图，例如："去修" / "fix it" / "修一下"

### 执行流程

```
Step 1  列出要修复的问题，按影响优先级排序
Step 2  展示修改 diff，用户可中止
Step 3  修改代码（最小化改动，不引入额外重构）
Step 4  重新验证：
        - verify 来的 → 重跑 verify
        - debug 来的 → 重跑 debug + 截图对比
Step 5  对比前后结果，确认问题消除
Step 6  仍有失败 → 回到 Step 2（最多 3 轮）
        3 轮后仍失败 → 停止，报告卡点，请用户介入
```

### 安全边界

- 只修改代码，不动数据库、不改配置文件、不部署
- 每次修改前必须展示 diff，用户确认后再执行
- 超过 3 轮自动停止，避免无限循环

## Report Engine 输出格式

### Verify 模式报告

```
## 验收报告 — <页面标题> (<url>)
验收时间：YYYY-MM-DD HH:mm

### 验收结果

| # | 验收项 | 结果 | 备注 |
|---|--------|------|------|
| 1 | 用户可正常登录 | ✓ | |
| 2 | 登录失败显示错误提示 | ✗ | 无任何提示，见截图2 |

通过：X/N  失败：Y/N

### 失败详情
**[N] <验收项名称>**
- 现象：<描述>
- 截图：[screenshot-N.png]
- 根因推断：<简短分析>

### 修复建议
1. <具体步骤>
```

### Debug 模式报告

```
## 排查报告 — <url>

### 根本原因
<一句话总结>

### 证据链
1. [Console] <错误信息> — <文件:行号>
2. [Network] <请求> → <状态码>，<说明>
3. [截图] <描述>

### 复现步骤
1. <步骤>

### 修复建议
1. <具体步骤>
```
